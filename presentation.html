Berkeley Orders Of Magnitude

Awkward things: IO -> implicit, late binding, no cleaning up of errors Concurrency/distributed computation -> the actual graph of nodes is specified elsewhere. original essential idea behind Actors model State -> local state is available at beginning of timestep. no way to write a program that relies on synchronized state (unless explicit about it). easy to recreate environment of a crash or bug. Errors -> fail fast/never write clean up code in core logic (what error path?)
